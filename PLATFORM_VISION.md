## 1. الرؤية العامة

بناء منصة رقمية متكاملة لإدارة المدارس الحكومية القطرية، مستدامة لأكثر من **20 سنة**، تقدّم خدمات احترافية بتجربة مستخدم سلسة وبسيطة، يتبنّاها الجميع: معلمون، إداريون، أولياء أمور، وطلاب.

## 5. المعايير التقنية الثابتة

### 5.1 الأداء
| المعيار | الهدف |
|---------|-------|
| تحميل الصفحة | < 2 ثانية |
| توليد التقارير PDF | < 5 ثواني |
| الـ uptime | ≥ 99.5% |
| الاستجابة للـ API | < 500ms |

### 5.2 الأمان
- ✅ PDPPL — قانون حماية البيانات الشخصية القطري رقم 13/2016
- ✅ QNSA — متطلبات الجهاز القومي للأمن السيبراني
- ✅ CSRF + CSP + HTTPS إلزامي
- ✅ تشفير البيانات الحساسة (Fernet)
- ✅ سجل خرق البيانات — إشعار خلال 72 ساعة
- ✅ DPO — مسؤول حماية البيانات معيّن رسمياً

### 5.3 إمكانية الصيانة
- أي مطوّر Django يستطيع الفهم والتطوير
- توثيق كل endpoint في OpenAPI
- كود نظيف بلا منطق معقّد في الـ views
- Services Layer — الأعمال في services.py

## 6. قواعد التطوير الذهبية (لا استثناء)

1. **لا كود يصل لـ main بدون Pull Request مراجع**
2. **لا migration بدون اختبار على staging أولاً**
3. **لا بيانات حساسة في Git** — `.env` دائماً في `.gitignore`
4. **كل تقرير أو وثيقة رسمية = PDF ثابت في `static/docs/`** — لا توليد ديناميكي
5. **قبل أي قرار تقني مؤثر — ارجع للمدير أولاً**
6. **الـ API أولاً** — كل ميزة جديدة تدعم API من البداية
