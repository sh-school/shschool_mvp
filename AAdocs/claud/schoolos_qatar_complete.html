PDPPL — قانون 13/2016 وحماية البيانات

التطبيق التفصيلي على SchoolOS | غرامات 1–5 مليون ريال

⚠️ تحذير قانوني حرج — قبل أي نشر إنتاجي

SchoolOS تعالج بيانات حساسة من الفئة الخاصة: بيانات أطفال + صحية. هذه الفئة تتطلب أعلى مستوى من الحماية بموجب PDPPL. مخالفتها = 1–5 مليون ريال + وقف المنظومة + مساءلة جنائية.

متطلبات PDPPL — ما الوضع في الكود؟
تشفير البيانات الصحية (حرج)

allergies, chronic_diseases, medications في HealthRecord + ClinicVisit — نص عادي. دوال encrypt_field/decrypt_field موجودة في models.py لكن غير مُطبَّقة.

مخالفة فعلية قائمة الآن
ConsentRecord — الموافقة الصريحة

Model ممتاز موجود في core/models.py لكن غير موصول في أي flow. قبل معالجة بيانات صحية الأطفال يجب الحصول على موافقة ولي الأمر.

غائب وظيفياً
DPIA — تقييم أثر حماية البيانات

وثيقة إلزامية قبل بدء معالجة أي بيانات حساسة. غيابها = غرامة مستقلة 1M ريال.

RoPA — سجل معالجة البيانات

AuditLog model ممتاز. signals.py يُغطي بعض العمليات لكن ليس StudentMark وConsentRecord وتغيير الصلاحيات.

⚙️ الباك إند — ما يجب إضافته
Celery + Redis — المهام الثقيلة خارج الـ request

إرسال SMS، توليد PDF، إشعارات الغياب — background tasks. NotificationService جاهز للتحويل. يُقلل response time من ثوانٍ إلى milliseconds.

أولوية عالية
JWT (djangorestframework-simplejwt) للـ API

يتيح PWA أولياء الأمور + تطبيق React Native مستقبلاً. Refresh tokens قصيرة الأجل (15 دقيقة).

Sentry + structlog — مراقبة الإنتاج

JSON logs قابلة للبحث. Sentry يلتقط كل خطأ مع context كامل (user, school, request). ضروري مع 126 موظف وبيانات حقيقية.

django-ratelimit — حماية من brute force

5 محاولات/دقيقة على /auth/login/ + 100 طلب/ساعة على /api/. يعمل مع الـ lockout الموجود.

CI/CD Pipeline (GitHub Actions)

pytest → black → isort → bandit (security) → Docker build → deploy. يمنع أي كود معيب من الإنتاج.

pgBouncer + select_related في كل list view

Connection pooling يُقلل load الـ database 70%+. select_related يُقلل queries من N+1 إلى 1-3.

django-ninja — API v2 مع OpenAPI تلقائي

Type hints + Swagger docs + validation أقوى. مثالي للـ API التي ستُستخدم مع PWA والوزارة.

Data Retention Policy + Soft Delete

فترة احتفاظ محددة لكل نوع بيانات + آلية حذف تلقائي. مطلوب PDPPL.
