logout CSRF · race condition في failed_login · بيانات حقيقية في Git · user enumeration — كلها مكتشفة من قراءة متأنية للكود وغائبة من النسخة الأولى.

🆕 جديد v2 — حرج #1
بيانات حقيقية لطلاب وموظفين داخل المشروع في Git
ملف data/new_students_full.csv يحتوي أرقاماً وطنية، أسماء، وأرقام هواتف لـ 742 طالباً حقيقياً. ملف data/2_Normalized_Staff_List.csv يحتوي 126 موظفاً. مجلد data/ غير موجود في .gitignore. أي git push يرفع بيانات شخصية حقيقية. انتهاك صريح لـ PDPPL.
✅ الإصلاح: أضف data/ إلى .gitignore فوراً. استخدم git filter-branch أو BFG لمسح السجل التاريخي. هذه أولوية #1 قبل أي نشر.
🆕 جديد v2 — حرج #2
ثغرة CSRF في logout_view — أي موقع يمكنه تسجيل خروجك
logout_view يقبل طلبات GET بدون تحقق. أي موقع خارجي يمكنه إرسال <img src="/auth/logout/"> فيُسجّل خروج المستخدم تلقائياً عند تحميل الصفحة. ثغرة استغلال فوري.
✅ الإصلاح (3 أسطر):
def logout_view(request):
    if request.method != "POST": return redirect("dashboard")
    logout(request)
    return redirect("login")
حرج #3
ثغرة أمنية: /api/ بدون مصادقة — core/middleware.py
EXEMPT = [..., "/api/"] — أي طلب لـ /api/ يتجاوز كل التحقق. بيانات 742 طالب حقيقي متاحة بدون login. مخالفة مزدوجة: PDPPL + قانون 14/2014.
✅ الإصلاح الجاهز: scripts/Fix/middleware_fixed.py موجود في المشروع — نسخه إلى core/middleware.py. 5 دقائق.
حرج #4
totp_secret يُخزن نصاً عادياً — التعليق "مُشفَّر" غير صحيح
الحقل totp_secret = models.CharField(...) مع التعليق "مُشفَّر" — لكن دالة encrypt_field() غير مستخدمة هنا. في حال تسرّب قاعدة البيانات، كل مفاتيح 2FA مكشوفة. هذا تناقض بين التعليق والكود، وادّعاء الأمان غير مبرر.
✅ الإصلاح:
# عند إنشاء TOTP:
user.totp_secret = encrypt_field(pyotp.random_base32())
# عند الاستخدام:
totp = pyotp.TOTP(decrypt_field(user.totp_secret))
حرج #5

✅ الإصلاح: سطر واحد في .env
FERNET_KEY=your-fernet-key-here
البنية جاهزة بالكامل (property getters/setters). لا تعديل في models.py.
حرج #6
عتبة الغياب الخاطئة قانونياً — check_absence_threshold
الكود: threshold=3 في 7 أيام. لكن قانون 25/2001 المادة 7 يستند إلى 10% من أيام الدراسة (≈19 يوماً من 190). العتبة الحالية اجتهادية بدون سند قانوني.
✅ الحل: استبدل threshold الصلبة بـ percentage_threshold = total_school_days * 0.10 + إضافة تعليق يستشهد بالمادة 7 من قانون 25/2001.
